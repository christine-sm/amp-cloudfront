Mobile Phone Usage > Desktop Usage

  • The web is accessed by mobile phones more often the desktop computers
  • On a global scale, most mobile phone users have limited cellular data volume and speed
  • Content is often not optimized for mobile devices

let vs. var

  • Both let & var can be redefined
  • Variables declared with var are function scoped
  • Variables declared with let are block scoped {}

function scopeExample() {
  // function scoped
}

if () {
  // block scoped
}

var example


function varExample(centimeters) {
  // function scope
  if (centimeters > 100) {
    var meters = centimeters / 100;
    var metersString = meters + ' meters';
  }
  console.log(meters);
  return metersString;
}
  • meters is defined
  • Temporary variable has leaked out of scope

let example


function letExample(centimeters) {
  let metersString;
  if (centimeters > 100) {
  // block scope
    let meters = centimeters / 100;
    metersString = meters + ' meters';
  }
  console.log(meters);
  return metersString;
}
  • meters is undefined
  • Temporary variable has not leaked out of scope

const

  • Also block scoped like let
  • The value of const is never reassigned
  • Does not necessarily mean than const is immutable
  • Properties of const can change
  • Use Object.freeze() if you want to make an immutable object

const exampleArray = [1, 2, 3];

const exampleObject = {
  month: 'December',
  days: 31
};

Arrow Functions

() =>

  • Does not rebind value of this; this bound to parent scope
  • Remember: traditional anonymous functions bind this to element
  • No need for var self = this anymore!

// ES5 Anonymous Function
var exampleButtonOne = document.querySelector('#exampleButtonOne');

exampleButtonOne.addEventListener('click', function() {
  console.log(this);
});

// ES6 Arrow Function
const exampleButtonTwo = document.querySelector('#exampleButtonTwo');

exampleButtonTwo.addEventListener('click', () => {
  console.log(this);
});

Arrow Functions (cont.)

  • Concise syntax
  • Implicit return for unbracketed functions

// Make it an ES6 arrow function!

function introduceMe (name, job) {
  return 'My name is ' + name + ' and I am a(n) ' + job;
}

Default Function Parameters

Function parameters can be assigned a default value 



const introduceYou = (name, job, vowel = false) => {
  let article;
  if (vowel) {
    article = 'an';
  } else if (!vowel) {
    article = 'a';
  }
  return `My name is ${name} and I am ${article} ${job}`;
}

...Spread

Allows an expression to be expanded in places where multiple arguments, elements, or variables are expected 


// Multiple Arguments Expected

function spreadFunction(x, y, z) { }

const args = [0, 1, 2];

spreadFunction(...args);

...Spread

Combine arrays more easily 


let arr1 = [0, 1, 2];
let arr2 = [3, 4, 5];

// Append items from arr2 into arr1 using ES5
arr1.push(arr2);

// What's wrong here?
					

arr1 = [0, 1, 2];

// Append items from arr2 into arr1 using ES5
Array.prototype.push.apply(arr1, arr2);

// Append items from arr2 into arr1 using ES6
arr1.push(...arr2);

...Rest

  • Represent an indefinite number of arguments as an array
  • Same syntax as Spread

// ES5 function using ES6 ...Rest
function multiply(multiplier, ...theArgs) {
  return theArgs.map(function (element) {
    return multiplier * element;
  });
}

// "Arrowfied" Function
const multiplyArrow = (multiplier, ...theArgs) => theArgs.map(element => multiplier * element);
